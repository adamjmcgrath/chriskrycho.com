Set up a .NET Core F♯ Suave Project with Paket

With some extra details on paket.dependencies, paket.references, and MSBuild

February 04, 2017Filed under Tech#dotnet#fsharp#software developmentMarkdown source

I just set up a Suave project in F♯ with Paket on .NET Core running on macOS for the first time. All of this information is out there on the internet, but none of it is all in one place. In particular, figuring out what content paket.references requires in order to sync paket.dependencies with .fsproj MSBuild files was not obvious! So here is a minimal process to go from zero to a working Suave project!

  1. Download and run the latest .NET Core installer. (I know, this isn’t at all the Mac or Linux way of doing things. Hopefully eventually this will just be brew install dotnet.)

  2. Install paket: brew install paket.

    $ brew install paket

  3. Create a new project:

    $ mkdir my-project
$ cd my-project
$ dotnet new -l F#

  4. Initialize Paket in the root of your project.

    $ paket init

    This will create a paket.dependencies file for you, which specifies the dependencies to install.

  5. Add Suave as a dependency by adding it to the paket.dependencies file:

    source https://www.nuget.org/api/v2
nuget suave

  6. Manually create a paket.references file. This is the file Paket uses to keep its things in sync with MSBuild projects. You need this to keep the .NET Core build tools in sync with Paket and therefore for building your project (if I understand correctly—still new here!).1

    For our purposes, the file just needs the names of the packages to keep in sync. Paket will use that in combination with the lockfile here is simple, so you can just do echo Suave > paket.references.

  7. Run paket install. This will download and install all the packages specified in your paket.dependencies file, along with all of their dependencies. The exact set of every dependency all the way down gets saved into your paket.lock file. Hooray for make reproducible builds! Since we up paket.references in step 6, this will also update your .fsproj file with the references it needs for Suave.

  8. Open

  1. .NET Core is transitioning away from the project.json format back to MSBuild, and so to .fsproj files. I’m actually not clear on whether this is related to the paket.references file or not: project.json presumably also had to be kept in sync, but I’m late to this party.↩
